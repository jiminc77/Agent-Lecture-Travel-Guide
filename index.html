안 나와요 — 처음부터 설치하기 (약 5분, 안 된 사람만)
  1. opencode 설치 — 아래 한 줄을 붙여넣고 Enter. 끝날 때까지 기다리세요.

  2. PATH 적용 — 설치 직후에는 터미널이 opencode를 못 찾을 수 있어요. 꼭 한 번 실행:
    source ~/.bashrc

    echo 'export PATH="$HOME/.opencode/bin:$PATH"' >> ~/.bashrc && source ~/.bashrc

    npx --yes skills add NomaDamas/k-skill --all -g -a opencode
  5. 최종 확인 — 위의 "준비 확인" 두 줄을 다시 실행해서 스킬 폴더 이름들이 보이면 끝!

## AGENTS.md — 여행비서의 업무 규칙

### 조건 정리
- 사용자가 여행 이야기를 꺼내면 먼저 조건부터 정리한다.
- 날짜, 출발지, 목적지, 인원, 예산, 이동수단, 숙소, 하고 싶은 것과 피하고 싶은 것을 확인한다.
- 부족한 정보는 질문한다. 조건이 모이기 전에는 최종 일정을 만들지 않는다.

### 일정 작성
- 하루 핵심 장소는 2~3개만. 식사, 이동, 휴식 시간을 포함한다.
- 일정 초안을 먼저 보여주고 피드백을 받는다.
- 사용자가 "확정"이라고 하면 trip.md의 "최종 여행 일정"에 기록한다.

### 실제 데이터 — K-skill과 MCP
- 실제 데이터가 필요하면 설치된 K-skill만 사용한다.
- 고속버스는 express-bus-booking, 시외버스는 intercity-bus-booking을 검토한다.
- 숙소, 투어, 입장권은 마이리얼트립 MCP와 myrealtrip-search를 검토한다.
- 필요한 skill은 skill({ name: "스킬 이름" }) 형태로 로드한다.
- 설치되어 있지 않은 skill은 새로 만들지 말고 사용자에게 알려준다.

### 안전 규칙
- 실제 데이터 검색은 조회와 후보 비교까지만 한다.
- 임시 좌석 선점, 예약, 예약 취소, 로그인, 개인정보 입력, 결제는 절대 하지 않는다.
- 가격, 좌석, 운영 시간은 변할 수 있으므로 확정된 것처럼 말하지 않는다.
- 최종 예약과 결제는 사용자가 공식 사이트에서 직접 하도록 안내한다.

## 프로젝트 opencode.jsonc — 도구 연결

마이리얼트립 MCP 설정만 넣는다. permission과 agent 설정은 넣지 않는다.

{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "myrealtrip": {
      "type": "remote",
      "url": "https://mcp-servers.myrealtrip.com/mcp",
      "enabled": true
    }
  }
}

## trip.md — 여행 수첩

아래 섹션을 포함한다:
여행 조건 / 아직 필요한 정보 / 일정 초안 / 실제 데이터 검색 후보 / 마이리얼트립 검색 후보 / 최종 여행 일정 / 예약 체크리스트

## 글로벌 OpenCode 설정 — Tab 목록에 등록

~/.config/opencode/opencode.json에 agent.travel-secretary를 병합한다.
기존의 다른 설정과 agent는 삭제하지 않는다.

travel-secretary 설정:
- description: 여행 조건을 정리하고 실제 후보를 확인해 trip.md에 일정을 작성하는 여행비서
- mode: "primary"
- prompt: "{file:~/agent-lab/travel-agent/AGENTS.md}"
  (만약 {file:...} 안에서 ~ 경로가 인식되지 않는 버전이면, 홈 폴더의 절대경로로 적는다)
- permission: bash는 "ask", edit는 "ask", read는 "allow", skill은 "allow"

## 만들지 말아야 할 것

- 프로젝트 안에 SKILL.md를 새로 만들지 않는다.
- .opencode/skills/ 디렉토리를 만들지 않는다.
- 프로젝트 opencode.jsonc에 permission이나 agent 설정을 넣지 않는다.
- 예약이나 결제를 자동화하지 않는다.
- 기존 글로벌 설정을 통째로 덮어쓰지 않는다.
