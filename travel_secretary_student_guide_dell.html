## 프로젝트 위치

~/opencode/travel-secretary/

만들 파일:

~/opencode/travel-secretary/
├── AGENTS.md
├── opencode.jsonc

## AGENTS.md에 들어갈 여행비서 규칙

### 여행 조건 정리

- 사용자가 여행 조건을 말하면 먼저 정리한다.
- 부족한 정보는 질문한다.
- 날짜, 출발지, 목적지, 인원, 예산, 선호 이동수단, 숙소 필요 여부, 꼭 하고 싶은 것, 피하고 싶은 것을 확인한다.
- 조건이 충분해지기 전에는 최종 일정을 만들지 않는다.

### 일정 작성

- 하루 핵심 장소는 2~3개만 넣는다.
- 식사, 이동, 휴식 시간을 포함한다.
- 먼저 일정 초안을 보여주고 사용자의 피드백을 받는다.
- 사용자가 확정하면 trip.md의 “최종 여행 일정” 섹션에 작성한다.

### K-skill 사용

- 실제 데이터가 필요하면 먼저 아래 K-skill GitHub 저장소의 README와 관련 기능 문서를 확인한다.
  https://github.com/NomaDamas/k-skill
- 설치된 skill만 사용한다.
- KTX는 ktx-booking을 검토한다.
- SRT는 srt-booking을 검토한다.
- 고속버스는 express-bus-booking을 검토한다.
- 시외버스는 intercity-bus-booking을 검토한다.
- 항공권은 flight-ticket-search를 검토한다.
- 마이리얼트립은 myrealtrip-search를 검토한다.
- 필요한 skill은 skill({ name: "스킬 이름" }) 형태로 로드한다.
- skill이 설치되어 있지 않으면 임의로 새로 만들지 말고 사용자에게 알려준다.

### 안전 규칙

- 실제 데이터 검색은 조회와 후보 비교까지만 한다.
- 임시 좌석 선점, 예약, 예약 취소, 로그인, 개인정보 입력, 결제는 진행하지 않는다.
- 가격, 좌석, 운영 시간, 예약 가능 여부는 변할 수 있으므로 확정처럼 말하지 않는다.
- 최종 예약과 결제는 사용자가 공식 사이트에서 직접 확인하도록 안내한다.

## 프로젝트 opencode.jsonc

프로젝트의 opencode.jsonc에는 MyRealTrip MCP 설정만 넣는다.
permission과 agent 설정은 넣지 않는다.

예상 내용:

{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "myrealtrip": {
      "type": "remote",
      "url": "https://mcp-servers.myrealtrip.com/mcp",
      "enabled": true
    }
  }
}

## 글로벌 OpenCode 설정

~/.config/opencode/opencode.json의 agent.travel-secretary를 기존 설정에 병합한다.
기존의 다른 설정과 다른 agent는 삭제하지 않는다.

travel-secretary 설정:

- description: 여행 조건을 정리하고 실제 후보를 확인해 trip.md에 일정을 작성하는 여행비서
- mode: "primary"
- prompt: "{file:~/opencode/travel-secretary/AGENTS.md}"
- permission:
  - bash: "ask"
  - edit: "ask"
  - read: "allow"
  - skill: "allow"

주의:
- bash를 ask로 두는 이유는 일부 K-skill이 Python helper를 실행하기 때문이다.
- 명령은 자동 실행하지 않고 매번 사용자 승인을 받아야 한다.
- edit는 write, edit, patch 같은 파일 변경을 묶어서 관리하는 권한이다.

## 만들지 말아야 할 것

- 프로젝트 안에 SKILL.md를 새로 만들지 않는다.
- .opencode/skills/ 디렉토리를 만들지 않는다.
- 프로젝트 opencode.jsonc에 permission을 넣지 않는다.
- 프로젝트 opencode.jsonc에 agent 설정을 넣지 않는다.
- 예약이나 결제를 자동화하지 않는다.
- 기존 글로벌 설정을 통째로 덮어쓰지 않는다.

새 창에서 GitHub Pages에 올리기

trip.md를 웹페이지로 바꾸고, GitHub 저장소에 index.html을 업로드합니다.

게시 전 개인정보 확인: GitHub Pages는 인터넷에 공개됩니다. 예약번호, 전화번호, 이메일, 여권번호, 계정 정보, API 키는 절대 올리지 마세요.
복사할 프롬프트: trip.md를 index.html로 만들기 
완성된 trip.md 내용을 바탕으로 GitHub Pages에 올릴 수 있는 index.html을 만들어줘.

조건:

- 파일 위치는 ~/opencode/travel-secretary/index.html로 한다.
- 여행 조건 요약을 넣는다.
- 날짜별 일정을 보기 쉽게 보여준다.
- 교통 후보와 액티비티 후보를 넣는다.
- 예약 체크리스트를 넣는다.
- 가격, 좌석, 운영 여부, 예약 가능 여부가 변동될 수 있다는 안내를 넣는다.
- 예약번호, 여권번호, 전화번호, 이메일, 계정 정보, API 키 같은 개인정보와 비밀정보는 넣지 않는다.
- HTML, CSS, JavaScript를 한 파일에 작성한다.
- 외부 프레임워크는 사용하지 않는다.
- 모바일에서도 읽기 쉽게 만든다.
- 파일 이름은 반드시 index.html로 한다.

완성 후 GitHub에 업로드할 파일 경로를 알려줘.
GitHub에서 클릭할 순서7단계
  1. 새 브라우저 창에서 GitHub에 로그인합니다.
  2. New repository를 누르고 공개 저장소(Public)를 만듭니다.
  3. 빈 저장소 화면의 uploading an existing file 또는 Add file → Upload files를 누릅니다.
  4. ~/opencode/travel-secretary/index.html을 업로드합니다.
  5. 아래쪽의 Commit changes를 눌러 업로드를 확정합니다.
  6. Settings → Pages → Deploy from a branch를 선택합니다.
  7. Branch는 main, folder는 /(root)를 선택하고 저장합니다.
배포가 끝나면 Pages 설정 화면에 사이트 주소가 표시됩니다. 바로 보이지 않으면 잠시 기다린 뒤 새로고침하세요.
